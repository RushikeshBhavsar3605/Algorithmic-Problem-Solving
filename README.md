## 🗄️ Repository Architecture

The vault is strictly partitioned into three main domains for instant retrieval:

### 1. The Pattern Index (Core Algorithms)

Structured strictly by algorithmic pattern, not just difficulty. This acts as my modular knowledge base for data structures and state machines.

- 🌲 `/graphs` & `/binary-search-tree` – Traversal and topological optimizations
- 🧮 `/dynamic-programming` – Nested by state type (e.g., `01-fundamentals`, `02-dp-grids`)
- 🪟 `/sliding-window-two-pointers` – Subarray and substring optimization
- 🔢 `/bit-manipulation` & `/maths` – Low-level bitwise operations
- 📚 `/heaps`, `/linked-lists`, `/stack-and-queues`, `/strings`, `/prefix-suffix`

### 2. The Execution Log (`/daily-challenges`)

A chronological logging of a 350+ day streak. Partitioned by Year and Month (`/2025/11`, `/2026/02`, etc.), this directory proves long-term consistency and the ability to adapt to randomized daily constraints without context-switching fatigue.

### 3. Stress Testing (`/leetcode-contests`)

Solutions engineered under live, high-pressure environments. This folder represents raw problem-solving velocity, edge-case identification, and rapid debugging under strict 90-minute time limits.
